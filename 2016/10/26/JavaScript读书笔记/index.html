JavaScript读书笔记--似懂非懂

发表于 2016-10-26   |   分类于 JavaScript读书笔记   |  

一个人有多大的野心，就会有多大的最作为；

下面是阅读“精通JavaScript”这本书做的笔记。

关于JS相关的知识都只是在项目中使用的时候才去学习，自己对于前端的知识了解的很少。在接下来的几天时间里面将
好好的学习JS的知识点。

JS的性能优势体现

  • 灵活的语法，表达力强
  • 支持编译运行
  • 事件驱动和非阻塞式设计

变量使用的一些注意事项

  • 如果变量只是声明而没有赋值，则该变量的值是undefined(无定义)；
  • JS是一种动态类型语言，变量的类型没有限制，可以赋予各种类型的值；
  • JS引擎的工作方式是，先解析代码，获取所有被声明的变量，然后在一行一行的运行。

这样所有变量的声明就会提前被提升到代码的头部，这就是变量提升；
这里需要注意下，变量的提升只对var声明的变量有效，全局的变量没有效果；

  • JS中也有区块，使用{}来进行使用，这个去看不构成单独的作用域。也就是说区块中的变量与外面的变量，属于同一个作用域；

JS中的相互转换

  • Number函数将字符串转换为数值，要比parseInt()函数严格的多，只要有
    一个字符无法进行转换，就会直接输出NaN；

  • parseInt按顺序解析字符，而Number函数整体转换字符串的类型；

    JS关于对象的练习

  • 对象的一些操作

    var obj={
width:'100'
};
obj.width+20

JS中少用switch case,多用对象结构

  • switch case 不使用大括号，不利于代码形式的统一。
  • 结构类似于goto语句，容易造成程序的混乱，使得代码结构混乱不坦，不符合面向对象编程的原则；
      function doAction(action){
    switch(action){
      case 'hack':
        return 'hack';break;
      case 'slash':
        return 'slash';break;
      case 'run':
        return 'run';break;
      default:
       throw new Error('Hello World!');
    }
  }
    对于上面的代码我们可以通过对象的形式来书写；
        function doAction2(action){
    var actions={
      'hack':function(){return 'hack';},
      'slash':function(){return 'slash';},
      'run':function(){return 'run';}
    };
    if(typeof actions[action]!=='function'){
      throw new Error('Invalid action.');
    }
    return actions[action]();
  }
    是不是感觉很整齐，代码也容易阅读；

    函数的作用域

作用域指的是变量存在的范围，JS只有两种作用域：一种是全局作用域，变量在整个程序中一直存在，所有地方都可以读取；另一种是函数作用域，变量只有在函数内部存在；
其实这里的作用域我们可以安装后端语言的作用域来理解就行；

  • 函数内部变量提升

这个可以这样理解，只要是var声明的变量，不管在那个位置，变量声明都会被提升到函数体头部；
就好比下面的代码一样；

    function f(x){
      if(x>100){
        var tmp=x-10;
      }
    }

变量提升之后，tmp就会在自己最近的作用域前进行声明；

    function f(x){
      var tmp;
      if(x>100){
        tmp=x-10;
      }
    }

arguments对象

JS容许函数有不定数目的参数，所以需要一种机制，可以在函数体内部读取所有参数。说白了这个arguments就一直在js的函数中保留着，
我们可以使用它。我们可以往函数中传递很多参数，不需要去管函数中是否有几个形参。

arguments对象包含了函数运行时的所有参数，arguments[0]表示第一个参数，以此例推。

   var fs=function(arry){
        console.log(arguments[0]);
        console.log(arguments[1]);
        console.log(arguments[2]);
      }

JS面向对象

所有的JS数据都可以被视为对象，简单的说就是对象是一种无序的数据集合，由若干个键值对构成；

        var obj={
            p:'Hello World'   //键值对的形式
        };

这样就定义了一个对象obj，这个对象内部包括一个键值对，P是键名，字符串是成员值，之间使用冒号隔开，如果键值对多的话需要使用逗号分隔；
对象的生成方法有三种，

        var a={};
        var b=new Object();
        var c=Object.create(null);

这里面第二种是通过构造函数来表示的，第三种适合继承的场合。

  • 注意事项：

对象的所有键名都是字符串若是数值就会被自动转换为字符串；
对象的每一个键名有称为属性，如果一个i额属性的值是函数，通常把这个属性称为”
方法”，可以像函数那样调用；

引用指向的只能是具体的对象，而不是另一个引用。JS里的结果是沿着引用链一直上溯到原来那个对象。

        var 0={
            p:function(x){
                return 2*x;
            }
        };
        alert(o.p(1));

JS规定如果行首是大括号，一律解释为语句(代码块)，如果要解释为表达式(对象)，必须在大括号前面加上圆括号；

  • for in 循环的注意点
  1. 遍历的是对象所有可遍历的属性，会跳不可遍历的属性；
  2. 它不仅仅遍历对象自身的属性，还遍历继承的属性；
  • 函数重载

判断传入参数数量的能力和判断传入参数类型的能力。JS的每个函数都带有一个仅在这个范围内作用的变量，称为参数(argument)
,它是一个包含所有传递给函数的参数的伪数组。。

typeof类型检查

下面有两个类型检查，都是比较常用的。

  1. 使用typeof进行判断
    if(typeof num=="string"){
             num=parseInt(num);                
         }
         if(typeof arr=="string"){
             arr=arr.split(",");
         }

  2. 使用构造函数CTOR来判断

    if(num.constructor==String){
             num=parseInt(num);
         }
         if(str.constructor==Array){
             str=str.join(',');
         }

    闭包

    其实这个也不是很难理解，但是要在代码中表示出来，没有大量的编码是不可能的。闭包意味这内层的函数可以引用存在于包围它的
    函数内的变量，即使外层函数的执行已经终止。这个特性非常强大和复杂。
